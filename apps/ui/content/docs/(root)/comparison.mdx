There are a lot of great ways to build a React UI in 2026. This page is an honest, side‑by‑side look at where **tinji ui** sits relative to other popular options, so you can pick the right tool for your project.

tinji ui is a copy‑paste component library built on **Base UI** and **Tailwind CSS**, distributed through the shadcn registry, and designed to be readable by both developers and AI agents.

## At a glance

|                       | tinji ui                          | shadcn/ui                              | Whop frosted-ui                  | Radix UI / Base UI                |
| --------------------- | --------------------------------- | -------------------------------------- | -------------------------------- | --------------------------------- |
| **What it is**        | Copy‑paste component library      | Copy‑paste component library           | Installable component library    | Unstyled headless primitives      |
| **Foundation**        | Base UI                           | Radix UI (Base UI option available)    | Radix UI                         | —                                 |
| **Styling**           | Tailwind CSS v4 + semantic tokens | Tailwind CSS + semantic tokens         | Radix Colors + own design system | Unstyled (bring your own)         |
| **Theming**           | CSS variables, light/dark         | CSS variables, light/dark              | Token-based theme + appearances  | None (you provide it)             |
| **Distribution**      | shadcn registry (own code)        | shadcn registry (own code)             | npm package (`@whop/frosted-ui`) | npm package                       |
| **You own the code**  | Yes                               | Yes                                    | No (versioned dependency)        | No (versioned dependency)         |
| **Accessibility**     | Inherited from Base UI            | Inherited from Radix/Base UI           | Inherited from Radix UI          | Built‑in (the whole point)        |

## Philosophy

|                          | tinji ui                                              | shadcn/ui                                | Whop frosted-ui                            | Radix UI / Base UI                |
| ------------------------ | ---------------------------------------------------- | ---------------------------------------- | ------------------------------------------ | --------------------------------- |
| **Primary goal**         | Opinionated, production-tested system on Base UI      | Unstyled-friendly starter you fully own  | Cohesive branded design system (Whop)      | Correct, accessible behavior      |
| **Opinionation**         | Medium–high (dense, modern defaults)                 | Low–medium                               | High (matches Whop's look)                 | None                              |
| **Customization model**  | Edit the source you copied                           | Edit the source you copied               | Override via theme tokens / props          | Style from scratch                |
| **Best for**             | Teams wanting a ready system they can still own       | Teams wanting maximum control            | Teams adopting Whop's aesthetic            | Teams building a bespoke system   |

## AI-native

A core focus for tinji ui is being legible to AI coding assistants. Components are written to be clear, predictable, and consistent so language models can understand and modify them reliably.

|                              | tinji ui                                  | shadcn/ui                          | Whop frosted-ui     | Radix UI / Base UI |
| ---------------------------- | ----------------------------------------- | ---------------------------------- | ------------------- | ------------------ |
| **Agent Skill provided**     | Yes (`npx skills add aussieljk/tinji-ui`) | Community skills / rules exist      | No first‑party skill | No first‑party skill |
| **MCP / registry for agents**| shadcn registry (machine-readable JSON)   | shadcn registry (machine-readable) | npm only            | npm only           |
| **Designed for LLM editing** | Explicit design goal                      | Readable, widely trained on        | Standard library    | Standard library   |

See the [Skills](/ui/docs/skills) page for what the assistant knows.

## Particles, patterns, and blocks

Beyond primitives, libraries differ a lot in how many ready-made patterns they ship.

|                          | tinji ui                       | shadcn/ui                          | Whop frosted-ui            | Radix UI / Base UI |
| ------------------------ | ------------------------------ | ---------------------------------- | -------------------------- | ------------------ |
| **Primitives / components** | ~68 components               | ~50+ components                    | Dozens of components       | The primitives only |
| **Pre-assembled patterns**  | ~500+ particles              | Blocks (auth, dashboards, etc.)    | Composed components        | None               |
| **API-integrated components** | Atoms (via [Cal.com](https://cal.com/atoms)) | No                  | Some product‑specific       | No                 |

tinji ui's [particles](/ui/particles) are pre-assembled compositions — forms, tables, command menus, date pickers — that combine multiple primitives into ready-to-use solutions you can still pull apart and edit.

## When to choose what

- **Choose tinji ui** if you want an opinionated, production-tested system built on Base UI that you fully own, with a large particle catalog and first-class AI assistant support.
- **Choose shadcn/ui** if you want the most widely adopted copy-paste workflow and maximum freedom to start from minimal, unopinionated components.
- **Choose Whop frosted-ui** if you're building inside or alongside the Whop ecosystem, or you specifically want its branded aesthetic as an installed package.
- **Choose Radix UI or Base UI directly** if you're building a fully bespoke design system and want only the headless, accessible behavior with no styling at all.

If you're coming from shadcn/ui or Radix UI, see the [migration guide](/ui/docs/radix-shadcn-migration) for a smooth transition.
