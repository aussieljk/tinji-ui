# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code. You can
find the full documentation for it [in our repository](https://github.com/changesets/changesets)

We have a quick list of common questions to get you started engaging with this project in
[our documentation](https://github.com/changesets/changesets/blob/main/docs/common-questions.md)

## How releasing works in this repo

1. Make your changes on a branch.
2. Run `bun changeset` and follow the prompts to describe the change and pick a
   semver bump (patch / minor / major). This writes a markdown file into
   `.changeset/`. Commit it with your PR.
3. Merge your PR into `master`.
4. The **Release** workflow (`.github/workflows/release.yml`) opens (or updates)
   a "Version Packages" PR that consumes the changesets and bumps versions +
   changelogs.
5. Merge the "Version Packages" PR. The Release workflow then runs
   `bun run release`, which builds and runs `changeset publish` to publish the
   changed public packages to npm.

Only the publishable packages are released:

- `@tinji/ui` (`packages/ui`)
- `@tinji/typescript-config` (`packages/typescript-config`)

All apps (`ui`, `www`, `origin`, `@examples/calcom`) are ignored in
`.changeset/config.json` so they never get versioned or published.
